Use this guide to connect Auth0 to Lovable using SAML.

## Prerequisites

- Auth0 tenant admin access
- Lovable workspace Owner or Admin

## Configure in Auth0

<Steps>
  <Step title="Create app and enable SAML2 Add-on" stepNumber="1">
    Auth0 Dashboard → Applications → Create Application → Regular Web Application → Create → Add-ons → enable SAML2 Web App.
  </Step>
  <Step title="Configure SAML2 Add-on" stepNumber="2">
    In <code>Settings</code>:
    - Set <code>Application Callback URL (ACS)</code> to https://auth.lovable.dev/__/auth/handler
    - Paste the <em>Settings Map</em> JSON shown below
    - Save
  </Step>
  <Step title="Settings Map (paste into SAML2 Add-on settings)" stepNumber="3">
    ```json
    {
      "mappings": {
        "email": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier"
      },
      "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress"
    }
    ```
  </Step>
  <Step title="Copy values from Add-on panel" stepNumber="4">
    After saving, copy:
    - <code>Issuer</code>
    - <code>SAML Protocol URL</code> (SAML SSO URL)
    - <code>Certificate</code> (X.509)
  </Step>
  <Step title="Configure it in Lovable" stepNumber="5">
    In Lovable → Settings → Workspace → Identity (SAML 2.0):
    - <code>Issuer</code> → Auth0 Issuer
    - <code>SAML SSO URL</code> → SAML Protocol URL
    - <code>Certificate</code> → X.509 certificate
  </Step>
</Steps>
